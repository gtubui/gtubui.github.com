Forms

 October 18th, 2021

In “Best practices for form design” by Salim Ansari, the author highlights 20 key points to creating a good form. The most useful points I found were: minimize input fields by only asking for relevant information, stick with one column, use multiple steps if you need more fields, ask easy questions first, highlight required fields, provide placeholder text below the input field rather within it, size fields accordingly, and minimize dropdown menus by using radio buttons.

Other useful points: make sure the form works with the tab button, avoid generic labels such as “submit,” avoid resetting/clearing buttons or fields because it may prevent someone from filling out the form, and make sure the form is accessible.

While creating the sketch for my first form, I definitely did some things that Ansari advised against in this article, such as using a bunch of dropdown menus, and having a generic “submit” button. I also used more than one column. With these tips in mind, I intend to improve on my form-making skills.
